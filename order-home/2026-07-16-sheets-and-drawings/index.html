Sheets and Drawings

Next, I’d like to build a feature that ends up allowing me ‘flip' a note card or main document card from its markdown/milkdown editor to two other types of editors:

  • Sheet backed by react-spreadsheet
    • There should be a toolbar like Excalidraw’s but allows the background of a set of cells to be chosen from a set of simple colors that would look good in each Order theme but selectable in a generic way. Users should also be able to add custom colors for cells from a simple pallet selector.
    • Text should, by default, continue past the cell like a typical spreadsheet. You achieve this with column-based z-indexing: each column's background layer gets a higher z-index than the column to its left, so a later column's opaque background naturally paints over the overflow text from earlier columns. Empty cells simply don't render a background layer, letting the text pass through. The toolbar should allow setting one or more cells to ‘collapse’ the continue past cell.
    • Basic formulas should be supported
  • Drawing backed by Excalidraw
    • Allows instant editing of local drawing file using excalidraw library

These should be two icons next to the terminal icon.

When clicked, a corresponding file of the exact same name as the note should be created if it doesn’t already exist with the following convention:

  • File Name.md <- Normal main document or note file in Order
  • File Name.sheet.html <- HTML-backed react-spreadsheet
  • File Name.excalidraw <- excalidraw file
  • File Name.md should now have a YAML frontmatter field called `view: [note, drawing, sheet] which will tell Order to view that by default.
  • Every time a user clicks a sheet, or drawing icon the view will switch and persist in the note by modifying the view frontmatter field accordingly.

More notes:

  • The spreadsheet and drawing should cleanly expand in full screen mode. By default more rows and columns should be added to the view to fill the space.
