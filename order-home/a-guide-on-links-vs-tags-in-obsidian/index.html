  • What Obsidian calls a link can be defined as: “A connection from one note to another note”. In Obsidian, a tag is: “A connection from a note to an idea”. 2022-06-30 (View Highlight)
  • Links auto-refactor by default, and tags do not 2022-06-30 (View Highlight)
  • When you change the name of a file within Obsidian, all links to that folder will automatically change to be pointing to the right place. 2022-06-30 (View Highlight)
  • Click behavior is very different for tags and links 2022-06-30 (View Highlight)
  • Clicking on a tag will open a search for all files that contain that tag. 2022-06-30 (View Highlight)
  • Clicking on a link will either: Open the file the link is pointing to. Create the file the link is pointing to, if it does not already exist. 2022-06-30 (View Highlight)
  • Tags are implicitly counters 2022-06-30 (View Highlight)
  • Tags can be hierarchical 2022-06-30 (View Highlight)
  • Links can have aliases 2022-06-30 (View Highlight)
  • A link can have multiple “names”, in the format of An Alias To The Note Name. This can be handy, and is not something tags can do. 2022-06-30 (View Highlight)
  • Instances of each tag are automatically counted up by the Obsidian tag pane! This means that ever tag automatically is also a visual counter of how often that tag appears in your vault. 2022-06-30 (View Highlight)
  • Tags can be defined in front-matter 2022-06-30 (View Highlight)
  • If you want to organize using links, then eventually you are going to encounter the idea of a “higher-order note”. You might have heard the term “MoC” used around the Obsidian community. This stands for “map of content”, and they are a kind of “higher-order notes”. A higher-order note is any note that exists to help you navigate to other notes. The naming isn’t 100% agreed on, so don’t let that confuse you. “Indexes”, “MoCs”, “Higher-Order Notes”… They’re all pretty similar. 2022-06-30 (View Highlight)
