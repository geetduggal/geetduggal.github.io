Tech Habits - Order App Updates to Inspire Your Note-Keeping Workflow

2026-07-19 — read on Medium

Style refinements, spacetime & cloud calendar sync, spreadsheets, drawings, masonry lists, and a Johny-Decimal toggle!

In my Medium profile, I have a mission statement that spiritually aligns with Simon Sinek’s ‘Just Cause’. A Just Cause has five elements:

  • It stands for something, not against something: It’s optimistic and hopeful.
  • It’s idealistic: It’s inspiring, bold, and ultimately unachievable.
  • It’s inclusive: It invites and inspires others to join.
  • It’s resilient: It can endure technological, cultural, political, and industry changes.
  • It’s service-oriented: It’s aimed at the benefit of others.

The run-on sentence in my profile has stayed something like this for years:

In my posts I aim to provide surprisingly simple tips on how to use tech to streamline your workflow for maximal enjoyment and creativity.

“Simple tips” for “your workflow” is decidedly not “here’s an advertisement for my home-grown app”.

Given that we are in a world where it is extremely easy to vibe-code your own bespoke app that serves your needs, my Just Cause feels even more relevant.

I have an app with many simple features that, in aggregate, services my needs. My hope is that sharing details about it to you all, there may be a few useful ideas that will translate to your setup.

In this new world, the real vehicle to disseminate utility may be in these exact words as opposed to a mainstream app or plugin. This is because you can now literally take ideas written in text and translate them to usable apps.

In other words, instead of sharing apps and code libraries, it may be better to represent modules of your bespoke app as a collection of prose remixed from others ideas, and with your own takes.

I think this is pretty damn awesome.

That being said, I think there’s still utility in publishing my app and code for two reasons:

  1. Source code can be useful for others, including AI tools for other bespoke apps
  2. Public binary and app-store downloads will allow playing with the tool and being inspired in a way that text won’t do justice

The innaugural post on Order corresponded to a v0.1.0 version of the app and today, I released the v0.1.2 version. The goal of the v0.1.* series is to make the app as useful to me as possible as a daily driver and battle test it deeply. Later in this post, I’ll describe a few features developed since the inaugural post that may inspire you.

Longer-term, I aim to build to a v0.2.0 that targets an easily-installable binary for the desktop and includes an app on the app store so you can see if any ideas are inspiring for you.

Style improvements

The pile is a stack of editable cards, newest on top, with a folder’s main document as the wide center column. A subtle change from the original is that cards sit apart by elevation instead of hard borders. I opted for a feeling of a a lifted background, a very thin border, and a soft shadow so it looks more like a minimal card.

The whole app still re-skins from one setting, and ⌘T cycles the themes as I find that it is convenient and puts me in different moods to browse, write etc.

Spacetime and the calendar

One spacetime.md at the vault root holds where my work lives (Areas, Categories, Notable Folders) and when things happen (events and seasons). The calendar reads straight from those notes: Day, Week, Month, Year, Season.

I can push an event to Google Calendar just by placing an email addresses at the end of a spacetime line. An address that matches a connected account hosts it, and the rest get invited.

2026-07-20 09:00-09:30 : Sprint sync       #[Map Pipeline v2] you@example.com dana@example.com
2026-07-21 14:00-15:00 : Roadmap planning  #[Map Pipeline v2] you@example.com sam@example.com

I bring my own Google OAuth client and can connect accounts and set a default in Settings.

Sheets

Any note flips to a spreadsheet (the icon next to the terminal button). Under the hood it is just a plain HTML table saved next to the note as Name.sheet.html, so I can read or edit it in any text editor:

<table class="order-sheet">
  <tr><td>Item</td><td>Estimate</td><td>Actual</td></tr>
  <tr><td>Sofa</td><td>1800</td><td>1750</td></tr>
</table>

The app allows adding formulas, colored cells, and text that overflows like a real sheet (these are stored minimally in the HTML). Fullscreen has the whole toolset, including cell drag: select a range and drag it to swap it somewhere else.

Drawings

The same typa flip gives an Excalidraw canvas, saved as a Name.excalidraw file. Good for architecture sketches and quick diagrams. It stays with the note.

Sheets and drawings in the pile

They are not only fullscreen. A folder shows its sheet and drawing notes as small live cards, so I can read a budget or glance at a diagram without opening anything.

Directory view

Every Notable Folder is just a directory, so I can flip a card over and browse it as one: the files inside, with sizes and times, sorted by name or date, and rename, move, or delete right there. Files dragged in from Finder land in the folder.

Integrated terminal

There is a real terminal inside the card too (⌘4, or the terminal item in the ⋯ menu). It opens in the folder’s own directory, so I can run git, grep, claude or a quick script right where the notes live without leaving the app.

Masonry lists

A third list type next to cards and lines. Set list: masonry and the note's bullets become boxes sized to their content, laid out in columns. Links and images inside the text render live, and I can drag any box to reorder. This is great for quick brainstorming lists or grocery lists on the phone.

Johnny-Decimal mode

A Settings toggle that puts Johnny.Decimal ids on everything: Areas as ranges (10–19), Categories as numbers (11), folders as 11.01. It rewrites spacetime.md, renames the directories, and fixes up the wikilinks and event tags. Toggle it off and the ids come back off. This is there partly to illustrate how easy it is to switch between decimals and non-decimals in Order for Johnny-Decimal since Order essentially borrows all concepts from Johnny-Decimal except the decimals.

But the bigger reason is that I will soon conduct a longer tiny experiment with decimals vs. without and I wanted an easy way to flip between them like I used scripts in the past.

Overall, while Order still has a lot of usability improvements, it's already been my daily driver for weeks. I now only occasionally use Obsidian and other editors. A big feature for me is the minimalism and the fact that it gets rid of a lot of ‘boilerplate’.

For example, when I use Obsidian, if I want to create a quick drawing or do a some napkin math in a spreadsheet like environment, I need to make sure not only that the appropriate plugins are installed, but also think about where to put files and what to name them. Because there are rather strict conventions in Order, the file names for notes, sheets, and drawings all share a prefix and you can go from writing to drawing to working on a grid in an instant.
